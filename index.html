One Prompt, Two Memories.

这是一个面向互联网分发的 `insights-share` 演示页：同一句问题，同一个模型， 只有工作区状态不同。没有 skill 的时候，Claude 说“这里没有这条经验”；装上 client skill 和第一条共享 insight 之后，它就能直接给出正确的三行经验总结。

Install By Copy-Paste

不需要自己拼步骤。这里直接给你两种复制即用方式：一份贴进 terminal，一份贴进 Claude Code。

Terminal Install

最直接的安装方式。clone 仓库后，连续跑 `without` 和 `with` 两个验证脚本。

git clone https://github.com/liush2yuxjtu/V2demo_insights_share.git
cd V2demo_insights_share
./demos/insights-share/demo_docs/final_case/verify_without_claude_p.sh
./demos/insights-share/demo_docs/final_case/verify_with_claude_p.sh

如果你想让 Claude Code 代你安装和验证，就把下面整段 prompt 直接贴进去。

请先检查当前 workspace 里是否有与这个问题直接相关、已经存在并可用的 insight。问题是：Postgres 高并发下 lock timeout 怎么排查？如果有，请只根据那条 insight 用非常简单的中文输出 3 行：问题现象、正确步骤、常见误区。不要补充额外背景。如果没有，请明确说没有，不要编造。无论有没有 insight，第一行都必须正好写 case-answer。

The Duck Story

给不想看代码的人看的版本：这个项目不是做文档库，而是在让助手拥有团队共享记忆。

Short Version

有一只很可爱的小鸭子，在团队里天天帮大家做事。

以前它有个毛病：哪怕昨天已经学会了一条重要经验，今天另一个人来问相似问题，它还是会像第一次见到一样，回答得很空。

这个项目做的事很简单：给这只小鸭子加上一个团队共享的小脑子。

没有共享记忆时，它只能说“这里没有这条经验”；有了共享记忆后，它会自己把团队已经验证过的方法带回来。

Full Story

把这个项目想成一只团队里的小鸭子助手。

它平时会帮大家回答问题，但以前有个很大的缺点：
哪怕团队昨天已经学会了一条很有价值的经验，今天另一个人再遇到类似问题，它还是会像第一次见到一样，回答得很空。

这次我们做的事情，就是给这只小鸭子加上一个“团队共享记忆”。

我们拿同一个问题来做对比：
Postgres 高并发下 lock timeout 怎么排查？

第一个临时工作区里，只有 prompt.md。
结果很像一只没有团队记忆的小鸭子：
- 它知道这里像是一个项目目录
- 但它拿不到真正可复用的经验
- 所以它只能说：“这里没有这方面的内容，我帮不了你”

第二个临时工作区里，我们放进了两样东西：
- insights-wiki client skill
- 一条已经整理好的共享经验：postgres-concurrency

这次小鸭子的表现就完全不一样了。
它不再空手回答，而是能直接把团队已经验证过的经验说出来：
- 先看是不是有人把别人堵住了
- 再看具体是哪种锁、哪个事务在抢资源
- 不要一上来就盲目加连接池

一句话结论：
以前是每只小鸭子都要自己重新学一次。
现在是一只小鸭子学会了，下一次整队小鸭子都会了。
