insights-share proposal

把个人 insight 升级成团队可共享的 wiki skill。服务端负责托管与管理，客户端只做连接和静默注入。

解决什么问题

生成 insight 很慢，用户不想每次重造经验。
团队共享时不能直接复制他人的上下文，需要高层经验的热加载。
用户不想手动管理 insight，应该在后台无感检索。

系统组成

insights-wiki-server
局域网托管、管理 UI、CRUD、热加载、检索 API。
insights-wiki
安装连接、静默检索、上下文注入、with/without 对比。
wiki 四层结构
type index / INDEX.md / item.md / raw logs。

验证方式

用 20 个用例做触发率验证，同时录制 without 和 with 两条证据链，最终产出 HTML、导出文本和 server/wiki 操作记录。

建议命令：

python3 demos/insights-share/demo_codes/demo_runner/bootstrap_demo.py
python3 -m insights_share.cli insights-wiki-server --start --ui --seed-demo
python3 -m insights_share.cli insights-wiki --install --server http://127.0.0.1:8765
python3 -m insights_share.cli insights-wiki --question "Postgres 高并发下 lock timeout 怎么排查"
